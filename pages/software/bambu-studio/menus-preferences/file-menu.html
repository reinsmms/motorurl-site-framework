Bambu Studio – File Menu

Bambu Studio · Bambu Lab A1 · No AMS Lite · Studio Version: 2.4.0.70

Overview

The File menu is where Bambu Studio handles the core project tasks: starting a new project, opening an existing one, saving your work, importing models or settings, and exporting files for printing or sharing. It is one of the most important menus in the program because it controls how your work enters and leaves Studio.

For most users, the File menu becomes part of the regular workflow very quickly. Open a project, import a model, make adjustments, save your changes, then export or send the result where it needs to go.

Main Commands

New Project

Creates a new empty project. Use this when you want to start fresh without bringing in a previous layout, plate arrangement, or settings setup.

Open Project

Opens an existing Bambu Studio project file. This is the normal choice when returning to work you already saved earlier.

Open Recent

Shows recently used project files so you can jump back into them without browsing through folders each time.

Save Project

Saves the current project in its present state. This is the main “save your progress” command and should be used often during editing or slicing changes.

Save Project As

Creates a new saved copy under a different name. This is useful when you want to preserve an earlier version before trying major changes.

Quit

Closes Bambu Studio. Before using it, make sure important changes have been saved.

Import

Import Model

Imports model files such as STL, 3MF, and other supported formats into the current project. This is one of the most common File menu commands because nearly every print starts with bringing in a model.

Import Configuration

Imports saved settings or configuration data. This can be useful when moving print settings, machine setups, or slicing-related presets between systems or project setups.

Practical Use

The import area is where the workflow usually begins. If you downloaded a model, received a project from someone else, or want to bring in settings from another setup, this section is the first stop.

Export & Preset Tools

Export G-code / Plate Output

Exports the sliced result for printing or use outside the current project session. This is the command used when you want the output file rather than just the saved project structure.

Export Plate as STL

Exports the current plate arrangement as an STL-style output. This can be useful in special workflows where you want the arranged geometry instead of only the original project.

Export Plate as 3MF

Exports the plate as a 3MF-based file. This can be helpful for sharing a prepared layout or preserving richer project information than STL alone.

Export Preset Bundle

Packages presets together for backup, transfer, or reuse. This is especially helpful when you have invested time tuning settings and want to move them to another setup or preserve them before experimenting.

MakerWorld / Sharing-Related Options

Depending on the current version and setup, File may also include options related to sharing, publishing, or working with the broader Bambu ecosystem. These items are worth checking when you want to move from private project work into sharing or community use.

Workflow Tips

Save Before Big Changes

Before changing a lot of settings, resizing parts, rearranging plates, or testing new slicing ideas, save the project first. That gives you a reliable return point if the experiment goes sideways.

Use Save As for Versions

If you are trying alternate layouts, support strategies, or profile ideas, use Save Project As to create separate versions rather than overwriting the original every time.

Keep Models and Projects Straight

A model file and a project file are not the same thing. Importing a model brings geometry into Studio, while saving a project preserves the working setup around that model.

Export Only After You’re Ready

Export is best treated as the final output step after the plate, settings, and slicing results look right. Saving the project first helps preserve the editable version.
